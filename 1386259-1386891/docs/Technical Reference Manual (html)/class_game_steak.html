Detailed Description

Class for handling the GameState after the dog has eaten a steak.

The GameSteak class is derived from the abstract base class, GameState. The class runs the game when in a similar fashion to GameActive, but with some modifications how collisions are handled and the rendering to the screen. This state only lasts a finite period of time before the game state is returned to being in GameActive. The game state can change if the game is won (all bones eaten), or the user pauses the game

Handle any collisions between the dog and game objects.

In the game steak state, all collisions between the dog and the game objects are handled. If the dog eats a bone the score is updated and if no bones are left, the game is won. If the dog eats a key, the key disappears and a section of the maze is unlocked. If the dog eats another steak, the game state is re-updated to be of type GameSteak. If the dog collides with a catcher, only that catcher respawns and the number of lives remains unchanged.
